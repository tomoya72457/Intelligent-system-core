# ADR-0002: 言語方針は日本語主体(識別子・コミット type は英語)

このファイルは、本テンプレートと派生プロジェクトの記述言語に関する確定判断(2026-07-09 壁打ち)を記録する。

- **日付**: 2026-07-09
- **状態**: 承認

## 文脈

利用者は日本語話者のソロ開発者で、読む主体は「本人+コーディングエージェント」。ドキュメント・コミット本文・エージェントへの指示・エラーメッセージを何語で書くかを決める必要がある。現行の主要エージェント(Claude Code / Codex / Gemini / Cursor)は日本語指示を問題なく解釈する一方、コード識別子やコミット type はツールチェーン(Conventional Commits のパーサ、lint、grep 性)が英語形式を前提とする。

## 決定

- **日本語**: 全ドキュメント(docs/・AGENTS.md・README)、コード内コメント、コミット本文、hooks・スクリプトのエラーメッセージ、PR・Issue 本文。
- **英語形式**: コード識別子(変数・関数・型名)、コミット type(feat / fix / chore / docs / refactor / test / ci)、ラベル名(agent:ready 等)、ファイル名。

## 検討した代替案(必須)

- **案B: 全部英語** — 却下。読む主体の大半が本人である個人開発で、母語より遅く曖昧な読み書きを常時強いるのは純損失。エージェントへの意図伝達も母語の方が精度が高い(誤解によるやり直しコスト減)。OSS として外部貢献者を集める段階になれば再検討すればよい(two-way door)。
- **案C: 全部日本語(識別子も)** — 却下。日本語識別子は lint・補完・grep・外部ライブラリとの一貫性で摩擦が大きく、Conventional Commits 等の機械可読規約とも整合しない。
- **案D: 混在自由(規定なし)** — 却下。ファイルごとに言語が揺れると、エージェントが出力言語を安定させられず、生成物の言語もぶれる。境界を明文化しておく方が決定的。

## 結果

- 本人とエージェントの双方が最小摩擦で読める文書層と、ツールチェーンに乗る機械可読層が分離される。
- `AGENTS.md` にこの規約を常時規範として記載し、`.gitmessage` が日本語本文+英語 type の型を提供する。
- トレードオフ: 非日本語話者への公開時の可読性は下がる。README 等の多言語化は必要になった時点で追加する(YAGNI)。

## Confirmation(実装後に追記)

本テンプレートの全生成物(docs/・hooks のメッセージ・CI の警告文)が本方針で記述されている。逸脱はレビューと `docs/conventions/docs-rules.md` の規約で検出する。

## 信頼度と再検討トリガー

- **信頼度**: 高(利用者要件に直結し、コストの非対称性が明確)
- **再検討トリガー**: 外部貢献者の受け入れを始めたら。エージェントの日本語指示解釈で実害のある劣化を観測したら。
