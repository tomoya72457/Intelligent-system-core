# intake 判定記録: 常時コンテキスト予算+構造リンター機械強制

このファイルは、常時読み込むコンテキスト(AGENTS.md)に行数・バイト予算を設け、`tools/check_structure.py` で機械強制する方針の較正記録である。関連する3層知識構造・生きた文書運用の判定もここにまとめる。

- **提案日**: 2026-07-09
- **slug**: context-budget-enforcement
- **判定**: **Adopt**(W 4.75)
- **再審査日**: 2027-07-09

## 提案

`AGENTS.md` に予算(ソフト150行・ハード200行・ハード24,576バイト)を設定し、散文の努力目標でなく `tools/check_structure.py` の機械チェックとして強制する。詳細は skills / rules / docs へ段階開示する。

## 適用可能性ゲート

制約なし。標準ライブラリのみの Python チェッカで実現。

## 可逆性(two-way door)

**可逆**。予算値はチェッカの定数で、緩める/外すのは1ファイルの編集。

## 賛成論(3点以上)

1. **長文脈の劣化は実証済み**: 関連情報が中間にあると精度が落ちる(Lost in the Middle)。長いだけで劣化する context rot も観測され、命令数が増えると遵守率が落ちる(IFScale)。常時コンテキストは短いほど良い。
2. **打ち切りの実在**: Codex は AGENTS.md を 32KiB で打ち切る。予算超過は「読まれない末尾」を生むだけなので、予算内に収めることが実利。
3. **実運用の削減実測**: ルールとスキルを分離して起動トークンを83%削減した実測(ペパボ)がある。段階開示は空論でなく効く。散文で「短く書こう」と促すより、リンターで止める方が確実(原則1)。

## 反対論(3点以上)

1. **予算がきついと情報が入らない**: 150行に収まらない重要規範があるのでは。→ 重要規範のみ AGENTS.md、詳細は `docs/` と `.claude/skills/` へ段階開示。ハードは200行で余裕を持たせ、ソフト150行は警告に留める。
2. **機械強制は柔軟性を奪う**: プロジェクトによっては予算超過が正当。→ 予算値はチェッカ定数で調整可能。ハードは配布時の安全弁であり、プロジェクト側で変更できる。
3. **LLM が台帳を全文再生成して壊す**: 大きな文書をLLMに再生成させると文脈崩壊(context collapse)する。→ `docs/conventions/docs-rules.md` で「台帳の全文再生成禁止・追記型」を規約化して対処。
