# スタック規約: docs(ナレッジリポジトリ)

このプロジェクトは Intelligent-system-core テンプレートの docs プロファイルで初期化されている。
文書全般の共通規約(予算・追記型台帳・LLM全文再生成禁止など)は `docs/conventions/docs-rules.md` が正本。
本ページはナレッジリポジトリ固有の**構造運用**だけを定める。検証ゲートは `make structure` のみ(`make check` がそれを含む)。

## 役割フォルダ(リポジトリの第一階層)

コンテンツはルート直下の「役割フォルダ」に分ける。役割=更新のされ方の違い。例:

| フォルダ例 | 役割 | 更新のされ方 |
|---|---|---|
| `topics/` | 進行中の主題(1トピック=1フォルダ) | 能動的に編集 |
| `notes/` | 未整理のメモ・下書き | 追記のみ。育ったら topics/ へ昇格 |
| `archive/` | 完結した主題 | 原則編集しない(移動のみ) |
| `docs/` | このリポジトリ自体の運用文書(テンプレート由来) | 規約変更時のみ |

- フォルダ名は用途に合わせて自由に決めてよい。ただし**役割フォルダには必ずエントリREADMEを置き**、`tools/check_structure.py` の必須READMEリストに登録して機械強制する(手順: `docs/conventions/structure-config.example.md`)。
- 第一階層に役割の不明なフォルダ・ファイルを増やさない(ルート許可リストが検出する)。

## ルータREADME規約(強化版)

- ルート `README.md` と各役割フォルダの `README.md` は**ルータ**にする: 「〜したい/〜を知りたい → このファイル」の表が本体。ファイル一覧の羅列にしない(一覧は git が持っている)。
- ルータは**行き先への1行要約+リンク**まで。内容の複製は禁止(正本は常に1つ)。
- 新しいフォルダ・重要ファイルを足したら、同じ変更内で親ルータに行を足す。`make structure` が README の存在までは強制するが、中身の鮮度は人間とレビューが守る。

## 1トピック=1フォルダ+cockpit README(personal-os 型)

- 1つの主題(プロジェクト・調査・継続タスク)につきフォルダを1つ作る: `topics/<slug>/`。
- 各トピックフォルダの `README.md` は **cockpit(操縦席)**: 目的 / 現在地 / 次の一手 / 主要ファイルへのリンク を10-20行で保つ。開けば30秒で再開できる状態が基準。
- 詳細(経緯ログ・素材・議事録)はトピックフォルダ内の別ファイルへ。cockpit は常に「今」を映す(古い経緯は下部か別ファイルへ追い出す)。
- 完結したトピックは cockpit に結論を1段落追記して `archive/` へフォルダごと移動する。
- フォルダの大規模な移動・改名(物理再編)は `docs/conventions/docs-rules.md` のプリフライト手順に従う(リンク切れの一括発生を防ぐ)。

## 書き方の既定

- 1ファイル=1主題。長大化したら分割し、元ファイルはルータ化する。
- 台帳系ファイル(一覧・記録)は追記型。LLM による全文再生成はしない(context collapse 防止。正本: `docs/conventions/docs-rules.md`)。
- 日付は `YYYY-MM-DD`。記録系ファイル名は `YYYY-MM-DD-<slug>.md`。
